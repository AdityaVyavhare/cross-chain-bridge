# 1. Project Overview

### Project Name

**BRT Cross-Chain Bridge**

### Objective

Build a **decentralized cross-chain bridge** that allows users to transfer **BRT tokens** between **EVM-compatible testnets** using a **two-way pegging mechanism**.

Initially support:

* **Ethereum Sepolia**
* **Polygon Amoy**

### Core Mechanism

Two-way pegging:

**Chain A → Chain B**

```
Lock Token → Validator verifies → Mint Token
```

**Chain B → Chain A**

```
Burn Token → Validator verifies → Unlock Token
```

The system ensures **1:1 token representation across chains**.

# 3. System Actors

## 1. User

A normal wallet user who wants to **bridge BRT tokens between chains**.

Capabilities:

* Connect wallet
* Approve token
* Lock tokens
* Burn tokens
* Track bridge status

---

## 2. Validator

Validator verifies bridge transactions and executes destination operations.

For now:

```
Single validator
```

The **current MetaMask account acts as validator**.

Validator responsibilities:

* Monitor bridge events
* Validate transactions
* Execute mint/unlock operations

## Required Functions

### Lock Tokens

Used when moving token from **Chain A → Chain B**

```
lockTokens(amount, destinationChain)
```

Steps:

1. Transfer tokens to bridge
2. Lock tokens
3. Emit event

Event:

```
TokensLocked
```

---

### Burn Tokens

Used when moving token from **Chain B → Chain A**

```
burnTokens(amount, destinationChain)
```

Steps:

1. Burn tokens
2. Emit event

Event:

```
TokensBurned
```

---

### Mint Tokens

Validator executes mint on destination chain.

```
mintTokens(user, amount, sourceTx)
```

Event:

```
TokensMinted
```

---

### Unlock Tokens

Validator unlocks tokens.

```
unlockTokens(user, amount, sourceTx)
```

Event:

```
TokensUnlocked
```
